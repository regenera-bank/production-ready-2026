1. **Account** - Conta bancária com saldo em centavos

8. **JWT** - JSON Web Token para autenticação stateless
9. **gRPC** - Remote Procedure Call de alta performance
10. **Event Sourcing** - Registro de eventos (parcial no Transaction Service)
11. **Circuit Breaker** - Proteção contra falhas em cascata
12. **Idempotency** - Garantia de execução única de operação
13. **Pessimistic Locking** - Trava de banco de dados para consistência
14. **Multi-AZ** - Múltiplas zonas de disponibilidade (AWS)
15. **Horizontal Pod Autoscaler (HPA)** - Escalabilidade automática K8s
16. **Rate Limiting** - Controle de requisições (100 req/min/IP)
17. **KYC** - Know Your Customer (compliance)
18. **AML** - Anti-Money Laundering (anti-lavagem)
19. **Blast Radius** - Raio de impacto de falha (isolamento)
20. **Cold Storage** - Armazenamento de dados históricos

**Padrão de Nomenclatura Identificado:**
- Valores monetários: **sempre em centavos** (`amountInCents`, `balanceInCents`)
- IDs: **UUID v4** em todas as entidades
- Status: **enums definidos** (active/suspended/closed, pending/completed/failed)

### PASSO 6: Regulamentação - Código que Implementa Normas BACEN/CVM

**Localização do Compliance:**

```typescript
// COMPLIANCE SERVICE (Port 3008)
// apps/services/compliance-service/src/compliance.service.ts

@Injectable()
export class ComplianceService {
  
  // BACEN - Resolução 4.753/2019 (Gestão de Risco)
  async checkKYC(userId: string): Promise<boolean> {
    // Verifica documentação completa
    // CPF válido, endereço, renda declarada
    const user = await this.getUserData(userId);
    
    if (!user.cpf || !this.validateCPF(user.cpf)) {
      return false;
    }
    
    // Consulta lista de sanções (OFAC, ONU)
    const isSanctioned = await this.checkSanctionsList(user.cpf);
    return !isSanctioned;
  }
  
  // BACEN - Circular 3.978/2020 (PLD/FT - Anti-Lavagem)
  async detectSuspiciousTransaction(
    transaction: Transaction
  ): Promise<boolean> {
    const rules = [
      // Regra 1: Transação > R$ 10.000 sem justificativa
      transaction.amountInCents > 1_000_000,
      
      // Regra 2: Múltiplas transações fragmentadas (smurfing)
      await this.detectSmurfing(transaction.userId),
      
      // Regra 3: Perfil de risco incompatível
      await this.checkRiskProfile(transaction),
      
      // Regra 4: Horário atípico (madrugada)
      new Date().getHours() < 6
    ];
    
    return rules.some(rule => rule === true);
  }
  
  // CVM - Instrução 617/2019 (Cadastro de Clientes)
  async validateInvestorProfile(userId: string): Promise<string> {
    const profile = await this.getInvestorData(userId);
    
    // Suitability: perfil conservador/moderado/agressivo
    if (profile.monthlyIncome < 5000_00) { // R$ 5.000
      return 'CONSERVATIVE';
    }
    
    if (profile.investmentExperience > 5) {
      return 'AGGRESSIVE';
    }
    
    return 'MODERATE';
  }
  
  // LGPD - Lei 13.709/2018 (Proteção de Dados)
  async anonymizeUserData(userId: string): Promise<void> {
    // Direito ao esquecimento (Art. 18, VI)
    await this.database.transaction(async (tx) => {
      await tx.update('users')
        .set({ email: 'anonymized@deleted.com', cpf: null })
        .where('id', userId);
      
      // Mantém dados de transação (obrigação legal)
      // mas desvincula do usuário identificável
    });
  }
}
```

**Normas Implementadas:**
- **BACEN Resolução 4.753/2019**: Gestão de riscos (KYC)
- **BACEN Circular 3.978/2020**: Prevenção lavagem de dinheiro (PLD/FT)
- **CVM Instrução 617/2019**: Cadastro de investidores
- **LGPD Lei 13.709/2018**: Proteção de dados pessoais

enum Environment {
  LOCAL = 'local',
  DEVELOPMENT = 'development',
  STAGING = 'staging',
  PRODUCTION = 'production'
}

export class EnvironmentService {
  private readonly env: Environment;
  
  constructor() {
    // Ordem de precedência:
    // 1. Variável NODE_ENV
    // 2. Hostname (EKS detecta automaticamente)
    // 3. Default: LOCAL
    
    this.env = (process.env.NODE_ENV as Environment) 
      || this.detectFromHostname() 
      || Environment.LOCAL;
  }
  
  private detectFromHostname(): Environment {
    const hostname = require('os').hostname();
    
    if (hostname.includes('regenera-prod-')) {
      return Environment.PRODUCTION;
    }
    
    if (hostname.includes('regenera-staging-')) {
      return Environment.STAGING;
    }
    
    return Environment.LOCAL;
  }
  
  isProduction(): boolean {
    return this.env === Environment.PRODUCTION;
  }
  
  // Configurações mudam por ambiente
  getDatabaseConfig() {
    return {
      host: this.isProduction() 
        ? process.env.RDS_ENDPOINT 
        : 'localhost',
      
      ssl: this.isProduction(), // TLS apenas em prod
      
      poolSize: this.isProduction() ? 20 : 5
    };
  }
}

### PASSO 9: Limites de Precisão - Tratamento de Decimais

**Estratégia de Precisão Financeira:**

```typescript
// packages/core/src/value-objects/money.value-object.ts

/**
 * DECISÃO ARQUITETURAL: Aritmética de Inteiros
 * 
 * PROBLEMA: JavaScript usa IEEE 754 (ponto flutuante)
 * 0.1 + 0.2 = 0.30000000000000004 ❌
 * 
 * SOLUÇÃO: Armazenar SEMPRE em centavos (menor unidade)
 * 10 + 20 = 30 ✅
 */

export class Money {
  private readonly amountInCents: number;
  private readonly currency: string;
  
  constructor(amountInCents: number, currency = 'BRL') {
    // Validação: apenas inteiros
    if (!Number.isInteger(amountInCents)) {
      throw new Error('Money must be in cents (integer)');
    }
    
    // Validação: não permite negativos (saldo nunca < 0)
    if (amountInCents < 0) {
      throw new Error('Money cannot be negative');
    }
    
    this.amountInCents = amountInCents;
    this.currency = currency;
  }
  
  // Aritmética segura
  add(other: Money): Money {
    this.assertSameCurrency(other);
    return new Money(this.amountInCents + other.amountInCents);
  }
  
  subtract(other: Money): Money {
    this.assertSameCurrency(other);
    const result = this.amountInCents - other.amountInCents;
    
    if (result < 0) {
      throw new InsufficientFundsError();
    }
    
    return new Money(result);
  }
  
  // Formatação para exibição
  format(): string {
    const reais = Math.floor(this.amountInCents / 100);
    const centavos = this.amountInCents % 100;
    return `R$ ${reais},${centavos.toString().padStart(2, '0')}`;
  }
  
  // Nunca use .toFixed() ou parseFloat() em dinheiro!
  // ❌ ERRADO: const valor = parseFloat("10.50")
  // ✅ CERTO: const valor = new Money(1050)
}
```

**Procurando por BigDecimal (Não Encontrado):**

O sistema **NÃO usa** `BigDecimal` ou bibliotecas de precisão arbitrária porque:
- JavaScript nativo não tem BigDecimal
- A estratégia de **inteiros em centavos** é suficiente para o domínio bancário brasileiro
- Limite máximo: `Number.MAX_SAFE_INTEGER` = 9.007.199.254.740.991 centavos = R$ 90 trilhões

**Implicações:**
- Transações > R$ 90 trilhões: **sistema falha** (aceitável, não há contas com esse valor)
- Precisão: **perfeita** até 2 casas decimais
- Performance: **ótima** (aritmética de inteiros é nativa)

// packages/auth/src/guards/jwt-auth.guard.ts

@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {
  
  canActivate(context: ExecutionContext) {
    // [1] Extrai requisição HTTP
    const request = context.switchToHttp().getRequest();
    
    // [2] Busca token no header
    const token = this.extractTokenFromHeader(request);
    
    if (!token) {
      throw new UnauthorizedException('Token not provided');
    }
    
    // [3] Valida token JWT
    try {
      const payload = this.jwtService.verify(token, {
        secret: process.env.JWT_SECRET
      });
      
      // [4] Anexa usuário na requisição
      request.user = {
        userId: payload.sub,
        email: payload.email,
        roles: payload.roles
      };
      
      // [5] Continua para o controller
      return true;
      
    } catch (error) {
      throw new UnauthorizedException('Invalid token');
    }
  }
  
  private extractTokenFromHeader(request: Request): string | null {
    const authHeader = request.headers['authorization'];
    // "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    return authHeader?.split(' ')[1];
  }
}

```typescript
// packages/logging/src/interceptors/logging.interceptor.ts

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const { method, url, traceId } = request;
    
    // [1] ANTES da execução do controller
    const startTime = Date.now();
    
    logger.info('Request started', {
      traceId,
      method,
      url,
      timestamp: new Date().toISOString()
    });
    
    // [2] Executa o controller
    return next.handle().pipe(
      // [3] DEPOIS da execução
      tap((response) => {
        const duration = Date.now() - startTime;
        
        logger.info('Request completed', {
          traceId,
          method,
          url,
          duration: `${duration}ms`,
          statusCode: 200
        });
        
        // [4] Métricas (Prometheus)
        this.metricsService.recordHttpRequest({
          method,
          path: url,
          statusCode: 200,
          duration
        });
      }),
      
      // [5] Se houver erro
      catchError((error) => {
        const duration = Date.now() - startTime;
        
        logger.error('Request failed', {
          traceId,
          method,
          url,
          duration: `${duration}ms`,
          error: error.message,
          stack: error.stack
        });
        
        // [6] Métricas de erro
        this.metricsService.recordHttpError({
          method,
          path: url,
          statusCode: error.status || 500
        });
        
        throw error;  // Propaga o erro
      })
    );
  }
}
```

export class TracingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler) {
    const span = tracer.startSpan(context.getHandler().name);
    
    return next.handle().pipe(
      finalize(() => span.finish())
    );
  }

// 2. CacheInterceptor (Response Caching)
@Injectable()
export class CacheInterceptor implements NestInterceptor {
  async intercept(context: ExecutionContext, next: CallHandler) {
    const key = this.generateCacheKey(context);
    const cached = await this.redis.get(key);
    
    if (cached) {
      return of(JSON.parse(cached));  // Cache hit
    }
    
    return next.handle().pipe(
      tap(async (response) => {
        await this.redis.set(key, JSON.stringify(response), 'EX', 60);
      })
    );
  }
}

// 3. TransformInterceptor (Response Formatting)
@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler) {
    return next.handle().pipe(
      map((data) => ({
        success: true,
        data,
        timestamp: new Date().toISOString()
      }))
    );
  }
}

3. **Metrics**: Performance é medida (Prometheus)
4. **Caching**: Respostas frequentes são cacheadas (Redis)

## 🎯 GENERALIZAÇÃO vs ESPECIALIZAÇÃO

### GENERALIZAÇÃO: Padrões Arquiteturais Aplicáveis a Qualquer Sistema

**Padrão 1: Microserviços + Event-Driven**
- **Aplicação Ampla:** E-commerce, SaaS, IoT platforms
- **Lição Universal:** Desacoplamento temporal via mensageria escala melhor que chamadas síncronas

**Padrão 2: Domain-Driven Design**
- **Aplicação Ampla:** Sistemas empresariais complexos (ERP, CRM, Healthcare)
- **Lição Universal:** Linguagem ubíqua + bounded contexts reduz ambiguidade

**Padrão 3: SAGA Pattern para Transações Distribuídas**
- **Aplicação Ampla:** Reservas (hotéis, voos), Supply Chain, Order Management
- **Lição Universal:** Compensação é preferível a 2PC (Two-Phase Commit)

**Padrão 4: Observabilidade (Logging + Metrics + Tracing)**
- **Aplicação Ampla:** TODOS os sistemas distribuídos
- **Lição Universal:** Sem observabilidade, debugging é adivinhação

---

### ESPECIALIZAÇÃO: Decisões Únicas do Domínio Bancário

**Especialização 1: Aritmética de Inteiros para Dinheiro**
- **Por quê:** Evitar erros de ponto flutuante (IEEE 754)
- **Limitação:** Requer conversão manual (centavos ↔ reais)
- **Alternativa:** bibliotecas como `bignumber.js` (mais lenta, mais precisa)

**Especialização 2: Lock Pessimista em Saldos**
- **Por quê:** Consistência absoluta (ACID) é mais importante que throughput
- **Trade-off:** Performance ~30% menor vs lock otimista
- **Justificativa:** Saldo errado = prejuízo financeiro real

**Especialização 3: Compliance Service Separado**
- **Por quê:** Regulação bancária muda frequentemente
- **Benefício:** Isolamento de lógica regulatória facilita auditoria
- **Custo:** Overhead de mais um serviço

**Especialização 4: Blockchain para Audit Trail**
- **Por quê:** Imutabilidade provável para auditores/BACEN
- **Realidade:** Blockchain privado (não descentralizado) = database glorificado
- **Justificativa:** Marketing + "prova" criptográfica

### Insight 2: "Real-Time Balance Streaming"
**Problema:** Cache de 10s causa inconsistência visual (saldo desatualizado).

**Nova Ideia:**
```typescript
// WebSockets + Redis Pub/Sub
@WebSocketGateway()
export class BalanceGateway {
  @SubscribeMessage('subscribe_balance')
  async handleSubscribe(client: Socket, accountId: string) {
    await this.redis.subscribe(`balance:${accountId}`);
    
    this.redis.on('message', (channel, message) => {
      client.emit('balance_updated', JSON.parse(message));
    });
  }
}
```
**Benefício:** Saldo atualiza em <100ms sem polling.
